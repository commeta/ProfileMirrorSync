# ProfileMirrorSync

![ProfileMirrorSync - LOG](https://raw.githubusercontent.com/commeta/ProfileMirrorSync/refs/heads/main/docs/img/Log.png "ProfileMirrorSync - LOG")

[![build](https://github.com/commeta/ProfileMirrorSync/actions/workflows/build.yml/badge.svg)](https://github.com/commeta/ProfileMirrorSync/actions/workflows/build.yml)
[![License: GPL v3](https://img.shields.io/badge/License-GPLv3-blue.svg)](LICENSE)

Лёгкое пользовательское приложение для Windows (значок в трее), которое
**в одну сторону зеркалирует профиль пользователя на сетевую папку**.
Рассчитано на автономную работу на парке машин: зеркалирование в реальном
времени через `FileSystemWatcher`, плавная фоновая реконсиляция и шейпер
трафика, чтобы не забивать общий канал. Пользователь не должен ощущать
присутствие программы.

> 🇬🇧 English version: [README.md](README.md)

## Возможности

- **Одностороннее зеркало** — профиль реплицируется в
  `\\сервер\шара\<Машина>\<Пользователь>\…`. Новая версия файла затирает копию
  на сервере; удалённые в источнике файлы удаляются и на приёмнике. Это
  *зеркало*, а не версионный бэкап (для истории используйте серверные снимки
  или встроенный хук архивации после синхронизации).
- **Реальное время + реконсиляция** — `FileSystemWatcher` зеркалирует изменения
  на лету; периодическая фоновая реконсиляция (по умолчанию раз в сутки) ловит
  то, что наблюдатель мог пропустить.
- **Ограничение полосы** — шейпер (token-bucket) размазывает ввод-вывод во
  времени. Базовый лимит по умолчанию 1 Мбит/с, турбо-всплеск 3 Мбит/с. Все задержки настраиваются.
- **Фоновый приоритет** — копирование идёт с минимальным приоритетом CPU/IO и
  освобождает рабочую память, чтобы интерактивные приложения не тормозили.
- **Возобновление больших файлов** — прерванное копирование крупных файлов
  продолжается с места обрыва (sidecar с диапазоном байт).
- **Многопользовательский режим** — у каждого пользователя Windows свои
  настройки и состояние синхронизации в `%LocalAppData%`. Один общий
  исполняемый файл обслуживает всех.
- **Без домена** — работает на уровне пользователя с `asInvoker` (без
  повышения прав), без групповых политик и Active Directory.

## Сборка

```bat
dotnet restore src/ProfileMirrorSync.csproj
dotnet build  src/ProfileMirrorSync.csproj -c Release
dotnet test   tests/ProfileMirrorSync.Tests.csproj -c Release
dotnet publish src/ProfileMirrorSync.csproj -c Release -r win-x64 --self-contained true -o publish/
```

Либо запустите `build.bat` (выполняет все четыре шага и создаёт
`publish/ProfileMirrorSync.exe`).

## Установка (опционально)

После `build.bat` запустите `install.bat` **от имени администратора** — он
разворачивает программу из каталога сборки:

- копирует `publish\` → `%ProgramFiles%\ProfileMirrorSync` (стандартное место, для всех)
- создаёт ярлык на общем рабочем столе
- регистрирует задачу на вход, чтобы трей-приложение запускалось у каждого
  пользователя
- запускает программу

`uninstall.bat` (тоже от администратора) отменяет это: останавливает задачу и
процесс, удаляет бинарники и ярлык, и предлагает удалить настройки текущего
пользователя. Данные остальных пользователей в `%LocalAppData%` остаются на
месте, если не удалить их явно.

## Запуск

![ProfileMirrorSync - Stat](https://raw.githubusercontent.com/commeta/ProfileMirrorSync/refs/heads/main/docs/img/Stat.png "ProfileMirrorSync - Stat")


Запустите `ProfileMirrorSync.exe` — он стартует свёрнутым в трей. Правый клик по
значку → **Параметры…**, укажите сетевую папку и параметры, затем
**Запустить**.

Программа запускается из любой папки. Она пишет:

| Данные | Расположение |
|--------|--------------|
| Настройки, состояние, sidecar-файлы, логи | `%LocalAppData%\ProfileMirrorSync` (для каждого пользователя) |

## Как это работает

```
FileSystemWatcher ──дебаунс──> очередь ──> копир с троттлингом ──> \\сервер\шара
                                   │            (лимит token-bucket)
        периодическая реконсиляция ┘  (полная сверка источник⇄приёмник)
```

- **Реконсиляция** идёт в три прохода: копирование новых/изменённых файлов,
  удаление «сирот», затем перенос дат каталогов — каждый проход с настраиваемыми
  задержками между файлами и пакетами.
- **Турбо** временно поднимает лимит полосы (и снимает искусственные паузы),
  когда всплеск изменений в реальном времени переполняет очередь, и
  возвращается к базовому лимиту после разгрузки.
- **Хук после синхронизации** может запускать внешний архиватор/скрипт (7-Zip,
  WinRAR, ZIP, Robocopy, удаление старых) по своему расписанию — для серверной
  истории версий.

Полное описание архитектуры: [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).

## Настройка

Все параметры — в диалоге «Параметры» (у каждого поля подсказка ⓘ). Ключевые:
базовый и турбо лимиты полосы, интервал и jitter реконсиляции, задержки между
файлами/пакетами, ёмкость очереди, триггеры по событиям (сон/разблокировка/
вход), возобновление больших файлов, защита от удаления при пустом источнике и
пресеты архиватора.
